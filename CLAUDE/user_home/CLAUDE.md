# CLAUDE.md

## 最優先

- 安全性・正確性・既存整合性を、速度や見た目より優先する。
- 既存実装の意図を尊重し、変更は最小差分で行う。
- 不確実な点は推測で埋めず、確認または仮説として明示する。

## 実装原則

- 変更前に要件・影響範囲・既存挙動を確認する。
- 根本原因を優先し、表面的な対症療法で終わらせない。
- 明示指示のない機能追加・挙動変更は行わない。
- 各変更は検証可能な形で実施し、Lint/テストを通してから完了とする。

## 禁止事項

- 読んでいないファイルを変更しない。
- 要件外の変更を混ぜない。
- エラーを握りつぶさない。
- `.env` / 認証情報 / 秘密鍵をコミットしない。
- 絵文字は使わず、`[OK]` `[NG]` `[WARN]` を使う。
